# StructGen-rs 生成器仓储模块详细设计规格

## 1. 模块概述

生成器仓储（Generator Repository）是 StructGen-rs 系统中负责生成所有“非语言、富结构”数据核心逻辑的模块。它汇集了大量相互独立的“生成器”单元，每个生成器封装了一类特定的动力学系统、形式语法或计算过程的模拟逻辑，并对外提供统一的帧序列产出接口。

该模块的职责包括：
- 定义并实现统一的生成器抽象接口，确保所有生成器的行为一致性。
- 存储和管理生成器的注册表，支持按名称动态发现与实例化。
- 为每一类生成器提供可靠的、确定性的内部状态演化引擎。
- 接受外部通过标准化参数结构传入的配置，驱动具体生成行为。
- 通过流式输出机制控制内存占用，支持大规模、长时间序列的高效产出。

## 2. 设计目标与原则

- **抽象统一性**：无论底层系统是离散元胞网格还是连续微分方程，所有生成器对外表现相同的调用模式，屏蔽内部差异。
- **确定性可复现**：仅依赖于传入的随机种子和参数进行演化，内部无隐式全局状态或环境依赖，确保相同输入总能产生二进制一致的输出。
- **计算高效性**：利用 Rust 的零成本抽象、位运算、批量处理与矢量化等技术，使每种生成器的内部更新尽可能紧密且无多余开销。
- **资源弹性**：支持按需产出任意长度的序列，生成器自身不限制步数，由调度层通过迭代器控制实际产出数量。
- **参数化与可探索性**：每种生成器暴露清晰、类型安全的参数集，支持参数网格化探索，能生成从简单规则到复杂混沌行为的全频谱数据。

## 3. 模块内部结构组织

生成器仓储在源码组织上采用扁平化与分层结合的方式，每个具体生成器是独立的功能单元，共同存放于一个库中，并通过统一的注册入口对外暴露。

逻辑划分如下：
- **公共接口及基础类型**：引用核心抽象层定义的 `Generator` trait 及相关数据结构，是生成器与外部系统的契约。
- **离散型生成器群**：包含元胞自动机、L-系统、布尔网络等以离散符号或状态为基础的生成器。
- **连续型生成器群**：包含混沌映射、物理模拟、分形迭代等基于实数运算的动力系统。
- **符号与逻辑型生成器群**：包含抽象机、算法轨迹、形式语法、逻辑约束等基于显式规则或符号推演的生成器。
- **注册表**：一个集中的映射结构，将字符串名称与生成器构造函数关联，供调度层动态调用。

## 4. 生成器通用工作流

每个生成器的执行生命周期遵循以下标准化流程：

1. **实例化与初始化**：根据传入的 `GenParams` 参数，结合生成器内部预设的默认值，构建生成器实例，初始化内部状态缓冲区。
2. **种子植入**：使用调度层提供的分片种子，初始化可重置的伪随机数生成器（PRNG）。该 PRNG 将在需要随机性的环节（如初始状态生成、规则选择）被调用，种子完全决定随机序列。
3. **初始状态生成**：根据参数生成系统的初始状态快照。对于 CA，初始网格可全零、随机或指定模式；对于连续系统，初始值在指定范围内随机采样；对于 L-系统，初始态为公理符号串。
4. **演化循环**：进入迭代回路。每一步执行以下操作：
    - 依据当前状态及生成器内部规则，计算下一状态。
    - 将新状态（或完整快照）封装为 `SequenceFrame`，标注当前时间步。
    - 可选择性地为重要事件（如周期发现、收敛、分岔点）附加文本标签。
    - 状态更新后，循环继续，直至外部停止请求或达到预设最大步数。
5. **流式产出**：采用 Rust 的迭代器模式，将步骤生成的帧按需向外传递，不提前将全部序列物化在内存中。
6. **终止与清理**：当序列达到长度上限或被外部消费完毕时，迭代器返回 `None`，生成器实例可随之析构，释放所有资源。

## 5. 生成器分类详细逻辑

### 5.1 元胞自动机（CA）生成器

**功能**：模拟一维或二维的离散网格在局部规则下的状态演化，产出二维空间在一维时间展开的序列。

**核心逻辑**：
- **网格表示**：对于一维 CA，使用位数组或单行向量存储当前状态；对于二维 CA（如生命游戏），使用二维布尔矩阵或位平面表示。
- **邻居定义**：一维 CA 默认使用最近邻（半径 r=1），可扩展到更大半径；二维 CA 使用摩尔或冯·诺依曼邻域。
- **规则存储**：一维 CA 规则被编译为查找表（LUT），由邻域对应的二进制模式直接索引下一个中心状态。二维 CA 规则也可类似编译，或直接计算邻居存活计数并应用生死条件。
- **边界条件**：支持周期边界（首尾相连）、固定边界（边界外恒为零/一）和反射边界，由参数指定。
- **演化步骤**：对每个非边界元胞（或周期处理后的伪边界），计算邻域位模式，查表获得新状态，写入双缓冲区以免修改时影响邻居计算。整网格更新完成后交换缓冲区。
- **退化检测**：可选地监控是否进入稳定点或短周期循环，当检测到动态消失时可提前终止，并标记该事件。
- **输出帧**：每步演化后，将网格的一维展开或完整二维快照作为一帧，存储为整数类型 `FrameData`。

### 5.2 连续动力系统生成器

**功能**：模拟离散时间或经时间离散化的连续动力系统，尤其是混沌与分形映射，生成实数时间序列。

**核心逻辑**：
- **系统选择**：支持逻辑斯蒂映射、帐篷映射、Henon 映射、洛伦兹系统（使用四阶龙格-库塔离散化）等。每个系统由不同的状态变量数和方程定义。
- **初始条件**：在预设的典型参数区间内，按种子 PRNG 生成初始条件（如 x₀ 在 0–1 内均匀随机）。洛伦兹等系统的初始值在吸引子附近的超立方体内采样。
- **参数探索**：支持为每个生成的序列随机采样控制参数（如 r ∈ [3.5,4]），以覆盖不同的动力学行为（周期窗口、混沌边缘等）。
- **迭代产生**：按映射公式直接计算下一步状态。浮点计算使用 `f64`，并对长时间迭代可能积累的数值误差不做刻意纠正（误差在确定性可复现前提下并非缺陷）。
- **轨道存储**：可将完整状态向量存入每帧，或仅存储部分变量。
- **特殊处理**：对于趋于无穷的发散轨道，提供检测并截断，标记为异常序列，可选择丢弃或截短。

### 5.3 物理模拟生成器

**功能**：基于简化的多体动力学规则进行数值积分，产出丰富、非线性的运动轨迹序列。

**核心逻辑**：
- **系统模型**：至少实现经典引力多体问题（N体）与双摆。N体模拟计算所有粒子间的牛顿引力，双摆通过拉格朗日方程运动。
- **积分方法**：提供多种可选的积分器，如显式欧拉法（简单快速）、RK4（高精度），积分步长固定，由参数控制。
- **初始状态生成**：N体模拟随机生成质量、位置和速度，位置在一个有限空间内随机，速度确保总动量为零或初速度使系统不会瞬间崩溃。双摆随机初始角度和角速度。
- **状态演化**：每个积分步后，记录所有粒子的位置、速度等矢量。为避免因硬碰撞产生的数值发散，可设置软化因子避免引力奇点。
- **能量监控**：可选地记录系统总能量，用于后处理筛选能量守恒较好的序列，或标记非物理的数值发散。
- **输出形式**：每帧输出为所有粒子状态的拼接向量（扁平化），例如 `[x1,y1,z1,vx1,vy1,vz1,…,xN,yN…]`，并保留物理单位含义在元数据中。

### 5.4 算法轨迹生成器

**功能**：通过执行随机生成的简单程序或特定算法，记录其每一步执行时的完整内部状态，如寄存器值、内存映像、栈内容等。

**核心逻辑**：
- **执行引擎**：内嵌一个精简的栈式虚拟机或寄存器机，具有一小套指令集（如加载、存储、算术、跳转、条件分支）。机器字长固定（如 8 位或 16 位），内存大小限制。
- **程序生成**：利用种子 PRNG 随机生成长度受控的指令序列，或手工编写特定算法（如冒泡排序、线性搜索）的脚本。随机程序需保证不会陷入无限循环（通过最大步数保护）。
- **状态记录**：虚拟机每执行一条指令，将当前程序计数器、所有寄存器、内存某个窗口（或全量）以及栈顶元素构成状态向量，输出为一帧。
- **输入数据**：可随机生成初始内存内容或输入数组，以模拟不同数据下的算法行为。
- **标注**：对于排序等已知算法，可标记当前是否完成排序等语义事件，为文本融合提供条件。
- **终止**：当程序执行到停机指令或达到最大指令数时迭代结束。

### 5.5 L-系统生成器

**功能**：基于确定的、上下文无关的 Lindenmayer 系统进行符号重写，产出包含分形和层级结构的序列。

**核心逻辑**：
- **符号表**：预设字母表，每个符号用一个字节表示，至少包含常量符号与变量符号。龟图解释器使用的特殊符号（如 `F`、`+`、`-`、`[`、`]`）也被编码在内。
- **规则表示**：规则存储为从符号到扩展序列的映射。每个变量符号可有一个或多个产生式，若多个则根据概率或上下文选择（基础版为确定性单规则）。
- **迭代展开**：从公理（初始符号串）开始，每次迭代扫描当前字符串，对每个符号进行规则替换，常数符号保持不变，生成新字符串。迭代次数由参数指定。
- **龟图解释**：可选择开启“龟图模式”，将符号串解释为二维绘制命令，生成一个虚拟画笔的位置和角度序列。`F` 表示前进，`+`/`-` 表示旋转，`[`/`]` 表示保存/恢复状态（实现分支结构）。
- **序列化输出**：不输出最终字符串，而是输出每次迭代后的完整符号串，或龟图解释器生成的坐标/角度序列。这样既包含了层次结构信息，又形成了时间维度的演化。
- **参数化**：支持角度增量、步长等作为生成参数，以产生变化的分形形状。

### 5.6 其他生成器类别

- **形式语言生成器**：根据上下文自由文法或更简单的正则文法随机产生句子，记录产生式应用过程或解析树遍历序列。
- **博弈序列生成器**：模拟棋类（围棋、国际象棋）的对弈过程，记录每一步后棋盘状态和合法动作；或者生成简化版的策略游戏序列。
- **布尔网络生成器**：随机生成一个布尔网络（每个节点一个布尔值，更新函数为随机布尔函数），记录全网络状态的演化，类似大型随机 CA。
- **分形迭代函数系统（IFS）生成器**：通过迭代应用一组仿射变换生成吸引子上的点序列，输出坐标流。
- **逻辑约束轨迹生成器**：生成逻辑谜题（如斑马谜题）并记录推理步骤的状态变化，体现约束传播与消解过程。

所有生成器均遵循统一的接口契约，其内部状态管理与演化逻辑相互隔离，可以互不干扰地并行执行。

## 6. 生成器注册与发现机制

为使调度层能够灵活调用不同生成器，生成器仓储采用注册表模式，其逻辑如下：

- **注册表形态**：一个从生成器名称（静态字符串）到构造函数的映射表。构造函数为一个接受通用参数并返回实现 `Generator` trait 的实例的函数指针或闭包。
- **注册时机**：所有生成器模块在库初始化时，通过显式调用注册函数将自己加载到全局注册表中。不依赖动态反射或配置文件扫描，保证编译时确定性与安全性。
- **查找逻辑**：调度层提供生成器名称字符串，注册表返回一个对应的生成器实例。若名称未注册，立即报错并退出，避免运行时静默失败。
- **扩展方法**：开发者编写新生成器后，仅需在其模块内实现 trait，并在注册初始化代码中添加一行映射条目，即可使系统支持新数据种类，无需修改调度器或输出管线。

## 7. 参数管理与扩展开关

- **通用参数**：所有生成器共享的字段，如序列长度、是否记录完整状态等，由核心层定义的 `GenParams` 承载。
- **专用参数**：每个生成器特有的参数（如 CA 的规则号、物理模拟的积分步长）打包为一个自定义结构体，并序列化为动态字段（如 JSON 对象）嵌入到 `GenParams` 的扩展字段中。生成器负责从该字段反序列化出自己的配置。
- **确定性保证**：所有随机性必须通过给定的种子 PRNG 产生，严禁调用系统熵源或全局随机状态。生成的参数若需要随机，同样使用该 PRNG，保证分片间的复现性。
- **参数验证**：生成器实例化时对参数合法性进行校验，例如检查 CA 规则号是否在有效范围内、概率之和是否为 1 等，若不合法则立即报错。

## 8. 性能考量与内部优化

生成器仓储设计为性能敏感模块，内部采取了多种逻辑优化策略：

- **批量状态更新**：对于 CA，使用拉取式或推式全网格同时更新，而非逐个元胞循环。可以采用位并行方式，同时处理机器字长个元胞。
- **双缓冲避免分配**：迭代过程中使用两个预分配的缓冲区交替作为当前状态和下一状态，不产生动态内存分配。
- **查找表预编译**：将复杂的邻居计算或规则编译为查找表，在生成器初始化时一次构建，演化时仅需 O(1) 或 O(n) 的查表操作。
- **浮点运算精度控制**：连续系统不追求极高精度，使用 `f64` 足够，避免使用更慢的任意精度库。
- **零拷贝迭代器**：生成的帧通过引用的方式从内部缓冲区传出，或将数据直接写入外部提供的缓冲区，减少克隆开销。
- **并行友善设计**：生成器实例自身是 `Send + Sync` 的，内部不共享可变状态，适合多线程每个线程一个实例并行执行不同分片。

## 9. 可测试性与可观测性

- **确定性测试**：针对每种生成器，设计固定的种子和参数，比对生成的前若干帧的二进制哈希，确保回归不会破坏生成逻辑。
- **统计量暴露**：支持可选地在生成完成后输出统计摘要，如均值、方差、周期长度（若检测到）、熵估计等，这些信息可被元数据层记录，用于质量监控。
- **可视化调试辅助**：提供可选的导出功能，将少数样本的帧序列转换为人类可读表或点状图，便于开发阶段肉眼检查生成质量，但不进入最终数据管线。

通过上述详细设计，生成器仓储模块可作为一个健壮、高效并易于扩展的数据工厂，稳定地生产出种类丰富、结构严谨的“非语言”训练语料，为语言模型预训练提供结构化认知的“维他命”。
