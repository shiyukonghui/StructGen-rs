## 概述

StructGen-rs 的元胞自动机生成器（CA2D、CA3D、BooleanNetwork）默认输出流式的 `SequenceFrame` 数据。为了将这些数据转换为适合模型训练的格式，我们需要使用后处理管道（pipeline）中的处理器。

### 训练数据格式要求

根据 [NCA数据格式对比分析](NCA数据格式对比分析.md)，Python 训练流程期望的数据格式为：

- **批量张量格式**: `(B, T, H, W, C)`
  - B: batch size（批量大小）
  - T: 时间步数（序列长度）
  - H, W: 网格高度和宽度
  - C: 通道数

- **Token 序列格式**: `[start][patches...][end]`
  - 每个 patch 通过 base-N 编码为单个 token
  - 多个帧串联成训练序列

## 处理器介绍

### 1. PatchTokenizer

**用途**: 将 2D 网格数据转换为 patch token 序列

**适用场景**: CA2D、NCA2D 等 2D 网格生成器

**配置参数**:

```yaml
- name: "patch_tokenizer"
  config:
    patch: 3              # 正方形 patch 边长
    num_colors: 2         # 颜色数 (= d_state)
    rows: 12              # 网格行数
    cols: 12              # 网格列数
    n_groups: 1           # 独立规则组数
    stride: null          # 滑动步长（默认等于 patch）
```

**编码算法**:
1. 将 (rows, cols) 网格切分为 (N_H, N_W) 个 patch
2. 每个 patch 做 base-N 编码：`token = Σ val[i] * (num_colors ^ i)`
3. 添加 start_token 和 end_token

**示例**:
- 12×12 网格，patch=3 → 16 个 patches
- start_token = num_colors^(patch²) = 2^9 = 512
- end_token = start_token + 1 = 513

### 2. SequenceStitcher

**用途**: 将多个独立帧的 token 序列串联成训练序列

**适用场景**: 所有生成器

**配置参数**:

```yaml
- name: "sequence_stitcher"
  config:
    frames_per_sequence: 10    # 每个序列包含的帧数
    add_sequence_start: true   # 是否在序列开头添加 start token
    add_sequence_end: true    # 是否在序列结尾添加 end token
    start_token: 10000         # 序列级 start token
    end_token: 10001           # 序列级 end token
```

**输出格式**:
- 不添加序列级 token: `[frame0_tokens][frame1_tokens]...[frame9_tokens]`
- 添加序列级 token: `[seq_start][frame0_tokens]...[frame9_tokens][seq_end]`

### 3. BatchCollector

**用途**: 将流式帧收集为批量数据

**适用场景**: 所有生成器

**配置参数**:

```yaml
- name: "batch_collector"
  config:
    batch_size: 5      # 批量大小（样本数量）
    num_frames: 10     # 每个样本的帧数
```

**输出格式**:
- 将多个样本收集为一个批量
- 每个 `SequenceFrame` 包含一个批量的所有数据

### 4. Normalizer

**用途**: 标准化数据值

**适用场景**: 所有生成器

**配置参数**:

```yaml
- name: "normalizer"
  config:
    max_val: 255       # 最大值（用于线性缩放）
    method: "Linear"   # 标准化方法
```

## CA2D 配置指南

### 基础配置

CA2D 是 2D 元胞自动机生成器，最适合使用 PatchTokenizer 进行处理。

```yaml
tasks:
  - name: "ca2d_training"
    generator: "ca2d"
    params:
      seq_length: 50
      extensions:
        rule_type: "lifelike"
        birth: [3]
        survival: [2, 3]
        d_state: 2
        rows: 12
        cols: 12
        boundary: "periodic"
        neighborhood: "moore"
        init_mode: "random"
    count: 10
    seed: 42
    pipeline:
      - name: "patch_tokenizer"
        config:
          patch: 3
          num_colors: 2
          rows: 12
          cols: 12
          n_groups: 1
      - name: "sequence_stitcher"
        config:
          frames_per_sequence: 10
          add_sequence_start: true
          add_sequence_end: true
          start_token: 10000
          end_token: 10001
      - name: "batch_collector"
        config:
          batch_size: 5
          num_frames: 10
    output_format: "Parquet"
```

### 参数选择建议

1. **网格大小**: 建议使用 12×12 或 24×24，能被 patch 大小整除
2. **Patch 大小**: 推荐 3，平衡 token 数量和信息保留
3. **序列长度**: 10-50 帧，根据训练需求调整
4. **批量大小**: 5-10，根据内存限制调整

### 不同规则配置

#### Conway's Game of Life

```yaml
extensions:
  rule_type: "lifelike"
  birth: [3]
  survival: [2, 3]
  d_state: 2
```

#### HighLife

```yaml
extensions:
  rule_type: "lifelike"
  birth: [3, 6]
  survival: [2, 3]
  d_state: 2
```

#### Day & Night

```yaml
extensions:
  rule_type: "lifelike"
  birth: [3, 6, 7, 8]
  survival: [3, 4, 6, 7, 8]
  d_state: 2
```

## CA3D 配置指南

### 基础配置

CA3D 是 3D 元胞自动机生成器，由于 PatchTokenizer 仅支持 2D，我们使用展平方式处理。

```yaml
tasks:
  - name: "ca3d_training"
    generator: "ca3d"
    params:
      seq_length: 30
      extensions:
        rule_type: "lifelike"
        birth: [5, 6, 7]
        survival: [5, 6]
        d_state: 2
        depth: 8
        rows: 8
        cols: 8
        boundary: "periodic"
        neighborhood: "moore"
        init_mode: "random"
    count: 10
    seed: 42
    pipeline:
      - name: "normalizer"
        config:
          max_val: 1
          method: "Linear"
      - name: "sequence_stitcher"
        config:
          frames_per_sequence: 10
          add_sequence_start: true
          add_sequence_end: true
          start_token: 10000
          end_token: 10001
      - name: "batch_collector"
        config:
          batch_size: 5
          num_frames: 10
    output_format: "Parquet"
```

### 3D 数据处理策略

由于 3D 数据无法直接使用 PatchTokenizer，有以下几种处理策略：

1. **展平策略**: 将 3D 网格展平为 1D 序列
   - 优点：简单直接
   - 缺点：丢失空间结构信息

2. **切片策略**: 将 3D 网格切片为多个 2D 平面
   - 优点：保留部分空间结构
   - 缺点：增加数据量

3. **直接使用**: 不使用 PatchTokenizer，直接输出展平数据
   - 适用于特定训练需求

### 不同规则配置

#### 3D Life (576)

```yaml
extensions:
  rule_type: "lifelike"
  birth: [5, 6, 7]
  survival: [5, 6]
  d_state: 2
```

#### Cyclic 3D

```yaml
extensions:
  rule_type: "cyclic"
  n_states: 4
  threshold: 1
```

#### Fredkin 3D

```yaml
extensions:
  rule_type: "fredkin"
  d_state: 2
```

## BooleanNetwork 配置指南

### 基础配置

BooleanNetwork 产生布尔值序列，使用基础处理器即可。

```yaml
tasks:
  - name: "boolean_network_training"
    generator: "boolean_network"
    params:
      seq_length: 50
      extensions:
        num_nodes: 32
    count: 10
    seed: 42
    pipeline:
      - name: "normalizer"
        config:
          max_val: 1
          method: "Linear"
      - name: "sequence_stitcher"
        config:
          frames_per_sequence: 10
          add_sequence_start: true
          add_sequence_end: true
          start_token: 10000
          end_token: 10001
      - name: "batch_collector"
        config:
          batch_size: 5
          num_frames: 10
    output_format: "Parquet"
```

### 参数选择建议

1. **节点数**: 16-64，根据训练需求调整
2. **序列长度**: 50-100 帧，观察网络演化
3. **批量大小**: 5-10，根据内存限制调整

## 最佳实践

### 1. 配置文件组织

建议按生成器类型组织配置文件：

```
tests/manifests/
├── gen_ca2d_training.yaml      # CA2D 训练配置
├── gen_ca3d_training.yaml      # CA3D 训练配置
└── gen_boolean_training.yaml   # BooleanNetwork 训练配置
```

### 2. 参数调优

#### 网格大小选择

- **CA2D**: 12×12 或 24×24（能被 patch 整除）
- **CA3D**: 8×8×8 或 16×16×16（适中大小）
- **BooleanNetwork**: 节点数 16-64

#### 序列长度选择

- **训练序列**: 10-50 帧
- **验证序列**: 5-20 帧
- **测试序列**: 1-10 帧

#### 批量大小选择

- **内存充足**: batch_size=10-20
- **内存有限**: batch_size=5-10
- **大规模训练**: batch_size=50-100

### 3. 输出格式选择

- **Parquet**: 推荐用于大规模训练数据
- **Text**: 用于调试和可视化
- **Binary**: 用于高性能数据加载

### 4. 性能优化

1. **并行生成**: 使用多个任务并行生成不同规则的数据
2. **流式处理**: 使用 pipeline 处理器减少内存占用
3. **批量收集**: 使用 BatchCollector 减少文件数量

### 5. 数据验证

生成数据后，建议进行以下验证：

1. **格式检查**: 确保输出文件格式正确
2. **数据完整性**: 检查数据是否完整
3. **训练兼容性**: 使用 Python 脚本验证数据可加载
