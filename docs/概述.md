## 3. Cargo.toml 构建配置
文件位置: Cargo.toml

### 运行时依赖
依赖 版本 特性 用途 serde 1 derive 序列化/反序列化 serde_json 1 - JSON 处理 serde_yaml 0.9 - YAML 清单解析 thiserror 1 - 错误类型派生 arrow 53 - Apache Arrow 列式内存格式 parquet 53 arrow Apache Parquet 列式存储 rayon 1 - 数据并行执行 sha2 0.10 - SHA-256 哈希校验 log 0.4 - 日志门面 crate tracing 0.1 - 结构化日志 tracing-subscriber 0.3 env-filter , json , fmt 日志订阅器 chrono 0.4 - 时间处理 clap 4 derive 命令行参数解析

### 开发依赖
依赖 版本 用途 tempfile 3 测试用临时目录

### 关键版本变化说明
开发规划文档中原始设计的是 arrow = "50" 和 parquet = "50" ，实际 Cargo.toml 中已升级到 53 。同时 chrono 在规划中标注了 features = ["serde"] ，但实际 Cargo.toml 使用的是默认特性。 tracing-subscriber 在规划中为 ["env-filter", "json"] ，实际增加了 "fmt" 特性。

## 5. 系统架构摘要
系统采用 分层架构 ，按自底向上的依赖关系分为五层：

```
   层                    模块                      说明
─────────────────────────────────────────────────────────
─────
  L1 核心抽象    core          FrameState/SequenceFrame/
  Generator trait/错误类型
  L2 独立功能    generators    9种生成器（CA、混沌系统、物理模
  拟、VM 等）
                 pipeline      6种后处理器（标准化、去重、差
                 分、令牌映射等）
                 sink          3种输出适配器（Parquet/Text/
                 Binary）
  L3 调度协调    scheduler     YAML清单解析、种子派生、分片、
  rayon并行执行
  L4 可观测性    metadata      进度追踪、元数据 JSON、
  tracing 日志
  L5 集成入口    main.rs       CLI参数解析(clap)、全流程装配
```
数据流方向 : 清单解析 -> 分片 -> (并行) 生成器 -> 后处理管道 -> 输出适配器 -> 文件系统 + metadata.json

## 6. 命令行接口
二进制 structgen-rs 支持的参数（ main.rs:L72-L105 ）：

参数 简写 必需 说明 --manifest -m 是 任务清单 YAML 文件路径 --output-dir -o 否 输出根目录（覆盖清单设置） --format -f 否 输出格式: parquet/text/binary --num-threads -t 否 并行线程数（默认CPU核心数） --log-level -l 否 日志级别（默认 info） --stream-write - 否 强制流式写出模式 --no-progress - 否 禁用进度条（CI 场景）

退出码: 0 =全成功, 1 =部分失败, 2 =完全失败/启动错误

## 7. 测试清单
在 tests/manifests/ 目录下有 28 个测试 YAML 文件 ，覆盖：

- 所有 9 种生成器的基础场景
- 3 种输出格式（Parquet、Text、Binary）
- 管道组合（单/双/全处理器链、裁剪、标准化）
- 错误场景（无效 YAML、重复名称、未注册生成器/处理器、空任务、零样本数、未知生成器/处理器）
- 多任务并行、格式覆盖
## 8. 总结
StructGen-rs 是一个 架构完整、设计文档齐全、代码实现完备 的 Rust 项目。7 个模块（core、generators、pipeline、sink、scheduler、metadata、CLI）全部有对应的设计文档和源代码实现。
