# 2D/3D 元胞自动机生成器扩展设计文档

> 版本：0.1 | 基于 StructGen-rs 现有架构 + nca-pre-pretraining 参考实现

---

## 1. 背景与目标

StructGen-rs 当前仅包含 1D 元胞自动机生成器（`ca.rs`，Wolfram 规则 0-255），而参考项目 `nca-pre-pretraining-main` 展示了 2D NCA 在 LLM 预预训练中的显著价值——164M NCA tokens 等效于 1.6B 自然语言 tokens。本设计将 CA 生成能力从 1D 扩展到 2D/3D，并引入神经网络规则系统，使项目能够生成更丰富的结构化时空数据。

**设计决策**：
- 独立新生成器模块（不修改现有 `ca.rs`）
- 支持 Game of Life 族 + Totalistic + NCA 三种规则系统
- 可配置多值离散状态（`d_state` 参数，默认 2 = 二值，可设 10 = 十色）
- 纯 Rust 实现，零新增 ML 框架依赖

## 3. 共享类型设计 — `ca_common.rs`

从现有 `ca.rs` 的私有类型中提取公共抽象，供三个新生成器复用。

### 3.1 Boundary（边界条件）

```rust
#[derive(Debug, Clone, PartialEq, Deserialize, Default)]

pub enum Boundary {
    #[default]
    Periodic,    // 环面拓扑，越界环绕
    Fixed,       // 越界视为状态 0
    Reflective,  // 越界镜像反射
}
```

### 3.2 Neighborhood（邻域类型）

```rust
#[derive(Debug, Clone, PartialEq, Deserialize, Default)]
#[serde(rename_all = "lowercase")]
pub enum Neighborhood {
    #[default]
    Moore,       // 全邻接：2D=8邻域, 3D=26邻域
    VonNeumann,  // 轴邻接：2D=4邻域, 3D=6邻域
}
```

### 3.3 InitMode（初始化模式）

```rust
#[derive(Debug, Clone, PartialEq, Deserialize, Default)]
#[serde(rename_all = "lowercase")]
pub enum InitMode {
    #[default]
    Random,        // PRNG 均匀随机填充
    SingleCenter,  // 全零，仅中心格子为状态 1
}
```

### 3.4 Rule2D（2D 规则系统）

```rust
pub enum Rule2D {
    /// B/S 记法：birth/survival 包含触发转移的邻居计数
    /// 仅适用于 d_state == 2
    LifeLike {
        birth: Vec<u8>,
        survival: Vec<u8>,
    },
    /// 邻居状态总和 → 下一状态
    /// 适用于任意 d_state >= 2
    Totalistic {
        transition_table: Vec<u8>,
    },
}
```

**编译优化**：Life-like 规则在工厂阶段编译为两个布尔查找数组 `birth_lut[0..max_neighbors+1]` 和 `survival_lut[0..max_neighbors+1]`，使演化时每个格子的规则评估为 O(1)。

### 3.5 Rule3D（3D 规则系统）

```rust
pub enum Rule3D {
    LifeLike3D {
        birth: Vec<u8>,
        survival: Vec<u8>,
    },
    Totalistic3D {
        transition_table: Vec<u8>,
    },
}
```

### 5.3 演化算法

```
fn evolve(src: &[u8], dst: &mut [u8], rows, cols, rule, boundary, neighborhood, d_state):

  对于每个格子 (r, c):
    1. 根据 neighborhood 选择偏移集
    2. 根据 boundary 计算邻居坐标
       - Periodic:   ((r+dr+rows)%rows, (c+dc+cols)%cols)
       - Fixed:      若越界则邻居值为 0
       - Reflective: 越界取边界值
    3. 计算邻居计数/总和
       - LifeLike: 统计活邻居数量（状态 > 0）
       - Totalistic: 求和 Σ neighbor_state
    4. 应用规则
       - LifeLike: 死格 → birth_lut[count], 活格 → survival_lut[count]
       - Totalistic: transition_table[neighbor_sum]
    5. dst[r*cols + c] = next_state
```

### 6.2 演化算法

与 Ca2d 相同的模式，扩展到 3D 索引：

```
索引计算: idx = d * (rows * cols) + r * cols + c

邻居偏移:
  Moore 3D: 26 偏移 {(-1,-1,-1), ..., (1,1,1)} \ {(0,0,0)}
  VonNeumann 3D: 6 偏移 {±1,0,0}, {0,±1,0}, {0,0,±1}

边界处理: 与 2D 相同逻辑，增加 depth 维度的环绕/固定/反射
```

### 7.2 神经网络权重

```rust
struct NcaWeights {
    conv3x3_kernel: Vec<f64>,  // [conv_features × d_state*n_groups × 3 × 3]
    conv3x3_bias: Vec<f64>,    // [conv_features]
    fc1_weight: Vec<f64>,      // [hidden_dim × conv_features]
    fc1_bias: Vec<f64>,        // [hidden_dim]
    fc2_weight: Vec<f64>,      // [d_state*n_groups × hidden_dim]
    fc2_bias: Vec<f64>,        // [d_state*n_groups]
}
```

**权重初始化**：从 `SeedRng` 顺序采样，`rng.next_f64() * 2.0 - 1.0`（均匀 [-1, 1]），匹配参考实现的 `nn.initializers.normal(1)`。每个种子 → 唯一权重 → 唯一 NCA 规则。

**参数量**（默认配置 d_state=10, n_groups=1）：
- 3×3 Conv: 9 × 10 × 4 = 360
- 1×1 Conv(1): 4 × 16 = 64
- 1×1 Conv(2): 16 × 10 = 160
- **合计：~584 参数 / 规则**

### 7.3 前向传播（纯 Rust，无外部 ML 框架）

```
输入: grid state [rows, cols, n_groups] (Vec<u8>)
  │
  ↓ Step 1: One-hot 编码
  state_oh [rows, cols, d_state × n_groups] (Vec<f64>)
  │
  ↓ Step 2: Wrap padding + 3×3 卷积
  padded = 环面扩展 1 格 (top↔bottom, left↔right)
  conv_out[k,r,c] = bias[k] + Σ(di,dj,ch) kernel[k,ch,di,dj] × padded[r+di, c+dj, ch]
  │
  ↓ Step 3: 1×1 卷积 (全连接层 1) + ReLU
  hidden[k,r,c] = ReLU(bias1[k] + Σ_j weight1[k,j] × conv_out[j,r,c])
  │
  ↓ Step 4: 1×1 卷积 (全连接层 2)
  logits[s,r,c] = bias2[s] + Σ_j weight2[s,j] × hidden[j,r,c]
  │
  ↓ Step 5: Identity bias 调整
  logits[s] += one_hot[current_state(s)] × identity_bias
  │
  ↓ Step 6: 采样
  temperature == 0 → argmax(logits)
  temperature > 0  → logits /= temperature; categorical_sample(softmax(logits), rng)
  │
  ↓
输出: next_state [rows, cols, n_groups] (Vec<u8>)
```

**选择纯 Rust f64 的理由**：
- NCA 网络极小（~584 参数，12×12 网格 ≈ 288K 运算/步），ML 框架调度开销超过计算本身
- 零新增依赖，保持项目轻量
- 纯 f64 运算跨平台确定性，匹配项目可复现性保证

### 7.4 Categorical 采样实现

```rust
fn categorical_sample(logits: &[f64], rng: &mut SeedRng) -> usize {
    // 数值稳定的 softmax
    let max_logit = logits.iter().cloned().fold(f64::NEG_INFINITY, f64::max);
    let exps: Vec<f64> = logits.iter().map(|&l| (l - max_logit).exp()).collect();
    let sum: f64 = exps.iter().sum();
    let cdf: Vec<f64> = exps.iter().scan(0.0, |acc, &e| { *acc += e / sum; Some(*acc) }).collect();

    let u = rng.next_f64();  // [0, 1)
    cdf.iter().position(|&c| u < c).unwrap_or(logits.len() - 1)
}
```

## 8. 统一输出格式

### 8.1 帧数据约定

所有三个生成器统一输出 `FrameState::Integer(i64)`（不同于现有 1D CA 的 `FrameState::Bool`），统一多值状态表达：

| 生成器 | 每帧 values 长度 | 值域 | 展开顺序 |
|--------|-----------------|------|----------|
| ca2d | rows × cols | [0, d_state-1] | 行主序 |
| ca3d | depth × rows × cols | [0, d_state-1] | depth→row→col 主序 |
| nca2d | rows × cols × n_groups | [0, d_state-1] | 行主序 × 组序 |

**帧大小**：
- ca2d 64×64 = 4,096 Integer/帧
- ca3d 16³ = 4,096 Integer/帧
- nca2d 12×12 = 144 Integer/帧

### 8.2 与管线的兼容性

| 处理器 | 兼容性 | 说明 |
|--------|--------|------|
| Normalizer | ✅ 完全兼容 | Integer 输入直接映射到 [0, max_val] |
| DedupFilter | ✅ 完全兼容 | 基于帧值去重 |
| DiffEncoder | ✅ 完全兼容 | Integer 差分 |
| TokenMapper | ✅ 完全兼容 | Integer → Unicode 码点 |
| ClipStitcher | ✅ 完全兼容 | 帧级截断/拼接 |

### 8.3 与输出适配器的兼容性

| 格式 | 兼容性 | 说明 |
|------|--------|------|
| Parquet | ✅ | state_dim 记录维度，值作为 Binary 存储 |
| Text | ✅ | 每个 Integer 映射为 Unicode 字符 |
| Binary | ✅ | 9 字节/值，state_dim 在文件头 |

  ├── registry.register("neural_cellular_automaton_2d", nca2d::nca2d_factory) // 新增别名
  └── ... (其他现有注册不变)

## 10. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 输出类型 | `FrameState::Integer`（非 Bool） | 统一多值状态表达；管线完全兼容 |
| 3D 网格尺寸 | 新增 `grid_size_3d` 字段 | 不破坏现有 `grid_size` 接口 |
| NCA 不引入 ML 框架 | 纯 Rust f64 数组运算 | 零依赖；小网络性能足够；跨平台确定性 |
| 规则复杂性过滤 | 管线处理器（未来扩展） | 关注点分离；可复用于任意生成器 |
| Patch tokenization | 管线处理器（未来扩展） | 同上；灵活；不硬编码 |
| 共享类型提取 | `ca_common.rs` 独立模块 | DRY；单一事实来源 |
| 双缓冲 `Vec<u8>` | 与现有 ca.rs 相同模式 | 已验证；迭代期间零堆分配 |
| 注册名策略 | 短名 + 长别名 | 匹配现有 ca/cellular_automaton 模式 |
| Seed = 规则（NCA） | 每个种子确定性产生唯一 NCA 权重 | 匹配参考实现；利用调度器种子派生 |
| 不修改现有 ca.rs | 遵循用户决策 | 风险隔离；后续可选择性重构 |

## 13. 性能分析

| 生成器 | 典型网格 | 格子/步 | 运算/步 | 估计吞吐(单核) |
|--------|----------|---------|---------|---------------|
| ca (1D) | 128 | 128 | ~640 | ~1.5M 步/秒 |
| ca2d | 64×64 | 4,096 | ~82K | ~12K 步/秒 |
| ca3d | 16³ | 4,096 | ~246K | ~4K 步/秒 |
| nca2d | 12×12 | 144 | ~288K | ~3.5K 步/秒 |

**内存占用**：双缓冲 + 帧输出 = O(网格尺寸)
- ca2d 64×64: ~8KB 双缓冲 + ~32KB 帧
- ca3d 16³: ~8KB 双缓冲 + ~32KB 帧
- nca2d 12×12: ~12KB 总计（含 one-hot 缓冲和权重）

**并行扩展**：Rayon 线程池线性扩展至多核。

## 14. 未来扩展点

### 14.1 管线处理器

**GzipComplexityFilter**：过滤低/高复杂性的生成序列
```yaml
pipeline:
  - name: "complexity_filter"
    params:
      method: "gzip"
      min_threshold: 0.4
      max_threshold: 1.0
```

**PatchTokenizer**：将 2D 网格帧编码为 patch-based token 序列
```yaml
pipeline:
  - name: "patch_tokenizer"
    params:
      patch_size: 2
      d_state: 10
      start_token: 10000
      end_token: 10001
```
