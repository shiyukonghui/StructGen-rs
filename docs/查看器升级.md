
根据对代码的分析，**目前项目的查看器（Viewer）不支持查看 NCA（神经网络元胞自动机）生成的数据**。

具体原因如下：

### 查看器的维度推断逻辑

在 [app.rs](file:///f:/RustProjects/StructGen-rs/src/view/app.rs#L319-L338) 的 `infer_dimension` 函数中，只支持三种生成器：

```rust
match args.generator.as_str() {
    "ca" | "cellular_automaton" => ...       // 1D
    "ca2d" | "cellular_automaton_2d" => ...  // 2D
    "ca3d" | "cellular_automaton_3d" => ...  // 3D
    _ => Err("unknown generator ...")         // ← NCA2D 会走这里
}
```

而 NCA2D 生成器在 `mod.rs` 中注册的名称是 `"nca2d"` 和 `"neural_cellular_automaton_2d"`，这两个名称都不在查看器的白名单中，所以会直接报错退出。

### 同理：参数构建也缺失

[build_gen_params](file://f:/RustProjects/StructGen-rs/src/view/app.rs#L260-L308) 函数也只匹配 `ca/ca2d/ca3d`，没有 NCA2D 的分支。NCA2D 有其特有的参数（如 `d_state`、`n_groups`、`temperature`、`hidden_dim`、`conv_features` 等），这些参数也没有被构建。

### 要支持 NCA2D 查看，需要修改什么

NCA2D 本质上产生的是 **2D 网格数据**，与 CA2D 的数据结构一致（`rows × cols` 的离散状态网格），所以改动量不大，需要在 `app.rs` 中添加对 `nca2d` / `neural_cellular_automaton_2d` 的处理：

1. **`infer_dimension`** — 返回 `CaDimension::TwoD`
2. **`build_gen_params`** — 传入 rows/cols 等参数，以及 NCA2D 特有的参数
3. **`infer_d_state`** — NCA2D 的默认 `d_state` 是 10（而非 CA2D 的 2），需要调整
